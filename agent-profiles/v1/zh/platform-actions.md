# Platform Actions（中文对照）

Source of truth:
[../en/platform-actions.md](../en/platform-actions.md)

本文档用于任务可能代表用户对本地宿主环境产生副作用时。

这份文档定义的是策略，不等于承诺这些能力当前一定可用。
某个动作是否真的可执行，取决于当前平台、已安装的 wrapper，以及项目本地集成是否存在。

例如：

- 写系统剪贴板
- 聚焦或启动桌面应用
- 打开浏览器或 URL
- 发送本地通知
- 在系统文件管理器中定位文件

## 默认原则

影响本地机器的 agent 动作应当满足：

- 范围窄
- 行为显式
- 尽量可回退
- 直接服务于当前任务

优先“替用户准备下一步”，而不是静默多做额外动作。

## 命令边界

如果已经存在稳定的高层 wrapper 命令，应优先使用它，而不是直接耦合底层传输细节。

优先选择满足以下条件的 wrapper：

- 隐藏平台相关 IPC 细节
- 输入输出清晰
- 失败可见
- 易于记录和验证

如果已经有稳定 wrapper 负责这一层，就不要把用户级策略绑定到具体 named pipe、二进制路径或传输编码上。

如果某个动作还没有稳定 wrapper 或明确归属的命令边界，默认应视为该动作当前不可用。

## Wrapper 发现规则

不要猜路径，而应使用显式 marker 文件。

默认契约：

- 读取 `~/.wezterm-x/agent-tools.env`
- 如果该文件不存在，就视为宿主侧 wrapper 当前不可用
- 不要从当前任务仓库、AGENTS 符号链接或其他无关环境细节反推 wrapper

对于剪贴板 wrapper：

- 从 `~/.wezterm-x/agent-tools.env` 读取 `agent_clipboard`
- 只有在该路径存在且可执行时才使用
- 如果缺少这个键，或者文件不可执行，就把剪贴板能力视为当前不可用

## 剪贴板

当输出明显就是给用户马上粘贴时，agent 可以主动写系统剪贴板。

典型允许场景：

- 一条短 shell 命令
- 一条 commit message
- 一小段代码
- 一个 URL
- 其他明显无 secret、预期会被粘贴到别处的短文本

默认限制：

- 除非用户明确要求，否则不要主动读取剪贴板
- 不要模拟粘贴，也不要依赖窗口焦点
- 不要在后台持续监听或同步剪贴板状态

以下情况先确认再写：

- secret 或凭证
- 破坏性命令
- 很长的多行脚本
- 非常大的 payload
- 很可能覆盖掉用户仍然需要内容的场景

写入剪贴板后，应明确告知用户剪贴板已更新，并概述写入内容。

## 其他宿主侧动作

只有在以下条件都成立时，agent 才应主动执行其他宿主侧动作：

- 该动作是当前任务的自然延续
- 风险低且容易理解
- 有稳定 wrapper 或清晰归属的命令边界
- 如果不做，用户本来也要手动完成同样的机械步骤

对于破坏性、持久性、隐私敏感或难以回退的动作，应先征求用户确认。

## 汇报

当宿主侧动作成功时，应汇报：

- 执行了什么动作
- 影响了什么目标
- 用户是否还需要继续做后续步骤

当动作失败时，应汇报失败的动作、已知的直接原因，以及主任务是否因此仍然受阻。
